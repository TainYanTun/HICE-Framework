# Data Dictionary: Myanmar Conflict Observatory

This document defines each column within the **Myanmar Conflict Observatory (MCO)** dataset. Our system utilizes a hybrid schema that combines raw forensic logs from **ACLED** with custom analytical layers developed for the **HICE (Health-Impact)** framework.

---

## 1. Core Forensic Columns (ACLED Source)

| Column Name | Description | Forensic Significance |
| :--- | :--- | :--- |
| `event_date` | The verified date the event occurred. | Primary axis for temporal trend analysis. |
| `event_type` | High-level classification (e.g., Battles, Explosions/Remote Violence, Strategic Developments). | Used to differentiate between direct kinetic engagement and non-combat events. |
| `sub_event_type` | Granular classification (e.g., Armed clash, Airstrike, Attack, Arrest). | Essential for identifying specific warfare modes like artillery or looting. |
| `actor1` | The primary group or entity involved in the event. | Used for raw actor identification. |
| `actor2` | The secondary group or entity involved (the opposing or associated force). | Identifies the interaction dynamics between groups. |
| `assoc_actor_1` | Groups allied with or supporting `actor1` during the event. | Used to track alliances (e.g., an EAO supporting a local PDF unit). |
| `assoc_actor_2` | Groups allied with or supporting `actor2`. | Identifies complex alliances on the opposing side. |
| `inter1` | A numeric code representing the *type* of group for `actor1`. | 1=State, 2=Rebel, 3=Militia, 4=Protester, 7=Civilian. |
| `inter2` | A numeric code representing the *type* of group for `actor2`. | Identical coding to `inter1`. (0 if no second actor). |
| `interaction` | A combined numeric code defining the relationship between `actor1` and `actor2`. | E.g., 12 (State vs. Rebel) is a combination of `inter1=1` and `inter2=2`. |
| `admin1` | The largest administrative division (State or Region). | Primary level for regional risk modeling. |
| `admin3` | The tertiary administrative division (Township). | Lowest level of administrative granularity for safety reporting. |
| `location` | The specific name of the village, town, or site. | Used for local-level event identification. |
| `latitude` / `longitude` | Precise geospatial coordinates. | Basis for all mapping and density visualizations. |
| `geo_precision` | A code (1-3) indicating the reliability of the location data. | 1 = Precise, 2 = Near, 3 = District-level centroid. |
| `fatalities` | The number of confirmed, verified deaths resulting from the event. | The primary metric for conflict lethality and the "Verified Floor." |
| `notes` | The full forensic narrative describing the event. | The "gold mine" for NLP extraction and HICE detection. |

---

## 2. Calculated & Analytical Columns (MCO Framework)

These columns are generated by the MCO Intelligence Engine during the transformation phase.

| Column Name | Description | Logic / Purpose |
| :--- | :--- | :--- |
| `actor1_clean` | Normalized version of `actor1`. | Maps 400+ groups into 6 stable categories (State, Resistance, EAO, etc.). |
| `actor2_clean` | Normalized version of `actor2`. | Enables simplified interaction network mapping. |
| `is_hice` | **Boolean Flag** (True/False). | Indicates if the incident meets the **Health-Impacting Conflict Event** criteria. |
| `hice_type` | Specific HICE classification. | Categorizes impact as *Infrastructure Damage*, *Personnel Targeting*, or *Access Disruption*. |
| `spotlight_name` | Pre-formatted string key. | Optimized for the **Humanitarian Spotlight Explorer** selector to ensure zero-lag loading. |
| `year_month` | Temporal grouping (YYYY-MM). | Used as the frame index for geospatial animations. |
| `actor2_viz` | Formatted version of `actor2`. | Ensures "N/A" or secondary actors are displayed cleanly in UI badges. |
| `assoc_actor_1_viz` | Human-readable allied actors for `actor1`. | Replaces empty values with "Sole Actor" for professional display. |
| `assoc_actor_2_viz` | Human-readable allied actors for `actor2`. | Replaces empty values with "Sole Actor". |
| `inter2_viz` | Descriptive label for `inter2`. | Translates numeric codes (e.g., 2) into text (e.g., "Rebel") for UI cards. |

---

## 3. Vulnerability Metrics (Tabular/Tab 7)

| Metric | Calculation | Strategic Value |
| :--- | :--- | :--- |
| **Severity Index** | Total Fatalities / Total Events | Measures the relative "lethality" of a region regardless of event volume. |
| **Vulnerability Score** | (0.7 * HICE) + (0.3 * Fatalities) | A weighted priority score used for evidence-based humanitarian triage. |

---

### Usage Note:
When exporting data from the **Data Records Explorer**, you will see these columns as part of the CSV/Dataframe structure. Internal columns (prefixed with `_` or specifically used for dashboarding) may be hidden in the UI but are available for full forensic audit in the codebase.
